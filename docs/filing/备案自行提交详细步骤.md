# 区块链信息服务备案 — 自行提交详细操作指南

**费用：¥0（自行提交，不走代理）**
**周期：20-30 个工作日**
**法律依据：《区块链信息服务管理规定》第十一条**

---

## 一、备案前准备（1-2 天）

### 1.1 确认备案主体

| 项目 | 要求 | 说明 |
|------|------|------|
| 备案主体 | 在中国境内注册的企业/机构 | 个人不可备案 |
| 营业执照 | 经营范围需包含"信息技术服务"或相关 | 如无，需先变更经营范围 |
| 法人身份证 | 法定代表人身份证正反面扫描件 | 需在有效期内 |
| 联系人信息 | 姓名、电话、邮箱 | 用于接收备案通知 |

### 1.2 准备材料清单

```
必须材料：
  ✅ 1. 营业执副本扫描件（PDF/JPG，≤5MB）
  ✅ 2. 法定代表人身份证正反面（PDF/JPG）
  ✅ 3. 区块链信息服务备案申请表（在线填写）
  ✅ 4. 服务名称：链聘通（D-HRS）
  ✅ 5. 服务类型：联盟链/私有链 — 人力资源数据存证
  ✅ 6. 服务器部署位置：[填写实际位置，如阿里云杭州]
  ✅ 7. 区块链底层平台：[填写实际平台，如 FISCO BCOS / 长安链 / 以太坊私有链]
  ✅ 8. 服务对象：企业 HR 部门和员工
  ✅ 9. 是否发币：否
  ✅ 10. 技术方案描述（已在 docs/filing/区块链信息服务备案材料.md 中准备）

可选材料：
  📎 11. 安全评估报告（可用 SECURITY_AUDIT.md）
  📎 12. 服务安全保障措施说明（可用 docs/compliance/ 中的文档）
```

### 1.3 在线注册账号

1. 打开全国互联网安全管理服务平台：`https://www.beian.gov.cn`
2. 点击"用户注册"
3. 选择"区块链信息服务提供者"
4. 填写企业信息，完成注册
5. 等待审核（1-3 个工作日）

---

## 二、在线填写备案申请（1-2 天）

### 2.1 登录并开始备案

1. 登录 `https://www.beian.gov.cn`
2. 点击"区块链信息服务备案"
3. 点击"新增备案"

### 2.2 填写服务信息

```
服务名称：链聘通（D-HRS）
服务类型：区块链信息服务
服务描述：基于区块链技术的人力资源管理系统，提供员工管理、
         凭证验证、薪资审批工作流、治理投票等服务。
         无加密货币支付功能，所有链上操作为审批存证。

区块链底层平台：[选择或填写]
  · 如使用 FISCO BCOS：选择"其他"，填写"FISCO BCOS"
  · 如使用长安链：选择"其他"，填写"长安链（ChainMaker）"
  · 如使用以太坊私有链：选择"以太坊"，填写"私有链部署"

节点数量：[填写实际节点数，如 3-7 个]
共识机制：[如 FISCO BCOS 默认 PBFT，长安链默认 TBFT]
智能合约语言：Solidity
```

### 2.3 填写技术方案

```
技术方案描述：

一、系统架构
D-HRS 采用"链上存证 + 链下存储"的混合架构：
- 链上：智能合约（Solidity 0.8.19）处理审批流程和事件记录
- 链下：Express.js API 服务器处理业务逻辑和数据存储

二、上链数据（仅存证）
- 员工信息哈希（SHA-256）
- 凭证签发记录哈希
- 薪资审批签名记录
- 治理提案和投票结果

三、不上链数据（链下加密存储）
- 个人身份证号（AES-256 加密）
- 具体薪资数额（AES-256 加密）
- 人脸生物识别信息（AES-256 加密）

四、安全措施
- 实名认证（KYC）：用户必须完成身份验证
- 制裁名单筛查：OFAC + 中国外交部名单
- 访问控制：基于角色的权限管理（RBAC）
- 数据加密：传输层 TLS 1.3，存储层 AES-256
- 内容审核：敏感词过滤
- 审计日志：所有操作记录

五、合规措施
- 无加密货币支付功能
- 无 ERC-20 代币发行
- 无钱包连接（MetaMask 等）
- 个人信息链下存储（符合《个人信息保护法》）
- 用户协议和隐私政策已建立
```

### 2.4 填写安全措施

```
  包含：访问控制、重入保护、速率限制、输入验证等

  已建立，详见 docs/compliance/10-安全应急响应预案.md

用户协议：
  已建立，详见 docs/legal/用户协议.md

隐私政策：
  已建立，详见 docs/legal/隐私政策.md
```

### 2.5 上传附件

```
上传顺序：
1. 营业执照副本扫描件
2. 法定代表人身份证正反面
3. 安全评估报告（可选，SECURITY_AUDIT.md 导出为 PDF）
```

## 三、提交并等待审核（20-30 个工作日）

### 3.1 提交前检查

```
检查清单：
  □ 服务名称是否正确
  □ 服务描述是否准确
  □ 是否发币：否
  □ 服务器位置是否填写
  □ 技术方案是否完整
  □ 安全措施是否说明
  □ 附件是否上传
  □ 联系人信息是否正确
```

### 3.2 提交后跟踪

1. 提交后系统生成备案号（如：京网信备 xxxx 号）
2. 登录平台查看审核状态
3. 如需补充材料，平台会发送通知
4. 审核通过后，获得备案编号

### 3.3 备案后义务

```
持续义务：
  1. 服务内容变更时，30 个工作日内更新备案
  2. 服务终止时，30 个工作日内注销备案
  3. 配合网信部门监督检查
  4. 建立用户投诉举报渠道
  5. 记录用户操作日志（≥6 个月）
```

## 五、常见问题

**Q1：没有企业实体怎么办？**
A：需要先注册公司。可以找代办公司，费用约 ¥1,000-3,000。

**Q2：经营范围不包含信息技术怎么办？**
A：需要先变更营业执照经营范围，增加"信息技术服务"或"软件开发"。

**Q3：服务器在境外怎么办？**
A：需要将服务器迁移至中国境内。推荐阿里云/腾讯云。

**Q4：备案被驳回怎么办？**
A：根据驳回原因修改材料，重新提交。常见原因：材料不清晰、描述不准确。

**Q5：备案后可以发币吗？**
A：不可以。备案只是信息服务备案，发币违反 924 通知。
