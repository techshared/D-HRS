### 3.2 系统架构
```
移动端 App (React Native) / Web 端 (HTML)
    ↓
API 网关 (Express.js + Helmet + HTTPS)
    ↓
业务逻辑层（员工管理 / 凭证存证 / 薪资审批 / 治理投票）
    ↓
链上存证层（Solidity 智能合约：EmployeeRegistry / CredentialRegistry / PayrollExecutor / HRGovernance）
    ↓
数据存储层（链上：哈希 + 签名 / 链下：加密数据库 MySQL/PostgreSQL）
```

### 3.3 安全措施
1. 实名认证：用户使用身份证 + 人脸识别完成实名认证
2. 权限控制：基于角色的访问控制（RBAC）
3. 数据加密：传输层 TLS 1.3，存储层 AES-256
4. 智能合约审计：已完成安全审计（详见 SECURITY_AUDIT.md）
5. 等保 2.0：正在进行三级测评

## 四、服务内容描述

### 4.1 核心功能
- 员工信息管理（注册 / 更新 / 状态变更）
- 可验证凭证（W3C VC）签发和验证
- 薪资审批工作流（多签审批，链上存证，实际发放由企业自行处理）
- 去中心化治理提案和投票
- 招聘、绩效评估、调薪、转岗、离职流程管理

### 4.2 上链数据
- 员工信息哈希（明文存储于链下数据库）
- 凭证签发记录哈希
- 薪资审批签名记录
- 治理提案和投票结果

### 4.3 不上链数据
- 个人身份证号（仅链下加密存储）
- 具体薪资数额（仅链下加密存储）
- 人脸生物识别信息（仅链下加密存储）
