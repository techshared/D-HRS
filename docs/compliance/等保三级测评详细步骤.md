# 等保 2.0 三级测评 — 详细操作指南

**费用：¥50,000-150,000（测评费）+ 技术整改费用**
**周期：3-6 个月（含整改）**
**法律依据：《网络安全法》第二十一条、第三十一条**

---

## 一、等保三级是什么

### 1.1 定义

等保（信息安全等级保护）是中国的网络安全国家标准。三级适用于：
- 涉及 >100 人个人信息的系统
- 金融、交通、能源等重要行业的系统
- 地市级以上政务系统

### 1.2 D-HRS 为什么需要三级

```
D-HRS 满足三级定级条件：
  · 处理 >100 人个人信息（员工姓名、身份证、薪资）
  · 系统中断会影响企业 HR 运营
  · 涉及链上数据不可篡改特性（特殊安全要求）
```

### 1.3 三级测评内容

```
73 个控制点，10 大类：

  1. 物理安全（6 项）— 机房安全
  2. 网络安全（6 项）— 防火墙、入侵检测
  3. 主机安全（6 项）— 服务器加固
  4. 应用安全（9 项）— 身份认证、审计
  5. 数据安全（4 项）— 加密、备份
  6. 管理制度（12 项）— 安全制度文档
  7. 管理机构（3 项）— 安全组织架构
  8. 人员安全（4 项）— 培训、考核
  9. 建设管理（5 项）— 开发安全
  10. 运维管理（18 项）— 日志、变更
```

## 二、详细步骤

### 第一阶段：定级备案（1-2 周）

#### 步骤 1：确定系统等级

```
定级依据：
  · 受侵害的客体：公民、法人和其他组织的合法权益
  · 受侵害程度：严重损害
  · 定级结论：三级

定级报告内容：
  1. 系统名称：D-HRS 链聘通
  2. 系统描述：基于区块链的 HR 管理系统
  3. 服务范围：企业 HR 部门和员工
  4. 数据类型：员工信息、凭证、薪资审批、治理投票
  5. 定级理由：处理 >100 人个人信息，系统中断影响 HR 运营
```

#### 步骤 2：公安备案

```
备案地点：所在地公安局网安支队
备案材料：
  1. 信息系统安全等级保护备案表
  2. 系统定级报告
  3. 系统拓扑图
  4. 安全管理制度清单
  5. 营业执照复印件

备案流程：
  1. 登录"公安机关互联网安全服务平台"
     网址：https://www.beian.gov.cn（或当地网安平台）
  2. 填写备案信息
  3. 上传材料
  4. 等待审核（5-10 个工作日）
  5. 获得备案证明
```

### 第二阶段：差距评估（2-4 周）

#### 步骤 3：选择测评机构

```
推荐测评机构（公安部授权）：

  机构名称              地区    费用范围      周期
  ─────────────────────────────────────────────────
  启明星辰              北京    ¥80K-120K    4-6 周
  绿盟科技              北京    ¥80K-120K    4-6 周
  天融信                北京    ¥60K-100K    4-6 周
  中国信息安全测评中心   北京    ¥100K-150K   6-8 周
  各省公安测评中心      各地    ¥50K-80K     4-6 周

选择标准：
  1. 是否在公安部授权名单中
  2. 是否有区块链系统测评经验
  3. 费用和周期是否合适
  4. 是否在当地有分支机构（方便现场测评）
```

#### 步骤 4：签订测评合同

```
合同内容：
  1. 测评范围：D-HRS 全系统
  2. 测评标准：GB/T 22239-2019 三级
  3. 测评费用：[与机构协商]
  4. 测评周期：[通常 4-6 周]
  5. 整改支持：是否包含整改指导
  6. 复测费用：首次不通过后的复测费用

合同签订后，测评机构会提供：
  · 差距评估清单
  · 整改建议书
  · 测评时间表
```

#### 步骤 5：差距评估

```
测评机构会检查以下内容：

技术类（43 项）：
  □ 网络架构是否安全（防火墙、IDS）
  □ 服务器是否加固（最小权限、审计日志）
  □ 应用是否安全（身份认证、输入验证）
  □ 数据是否加密（传输、存储）
  □ 备份是否到位（每日备份、异地存储）

管理类（30 项）：
  □ 安全管理制度是否建立（12 个文档）
  □ 安全组织机构是否设立
  □ 人员安全培训是否开展
  □ 应急响应预案是否建立
  □ 安全审计是否执行

当前 D-HRS 差距评估结果（参考 docs/compliance/等保三级差距评估自查清单.md）：
  · 管理制度：100%（12/12 文档已建立）
  · 物理安全：100%（云平台兜底）
  · 应用安全：56%（5/9 已满足）
  · 数据安全：25%（1/4 已满足）
  · 网络安全：0%（0/6 已满足）
  · 主机安全：0%（0/6 已满足）
```

### 第三阶段：技术整改（4-8 周）

#### 步骤 6：按差距清单整改

```
第一优先（2 周内完成）：

  1. 部署云 WAF
     · 阿里云：Web 应用防火墙（¥3,000-10,000/年）
     · 腾讯云：Web 应用防火墙（¥3,000-10,000/年）
     · 配置：防 SQL 注入、XSS、CC 攻击

  2. 配置安全组策略
     · 仅开放必要端口（443、3001）
     · 禁止管理端口公网访问
     · 配置 IP 白名单

  3. 启用系统审计日志
     · Linux：启用 auditd
     · 配置日志转发到集中日志平台
     · 日志保留 ≥ 180 天

  4. 实施数据库每日备份
     · 使用云数据库自动备份功能
     · 备份保留 30 天
     · 异地备份（不同地域）

第二优先（2-4 周内完成）：

  5. 部署入侵检测系统（IDS）
     · 阿里云：安骑士/云安全中心（¥5,000-15,000/年）
     · 配置告警规则

  6. 启用双因素认证（TOTP）
     · 管理员账号必须启用 2FA
     · 推荐 Google Authenticator 或阿里云 MFA

  7. 安装主机安全 Agent
     · 阿里云：云安全中心 Agent
     · 腾讯云：主机安全 Agent
     · 功能：漏洞扫描、入侵检测、合规检查

  8. 实现统一输入校验
     · 后端 API 统一参数验证
     · 防 SQL 注入、XSS

第三优先（4-6 周内完成）：

  9. 安全域网络隔离
     · Web 层 → 应用层 → 数据层 三级隔离
     · 使用 VPC 子网划分

  10. 日志分析告警系统
      · ELK Stack 或阿里云 SLS
      · 配置异常行为告警

  11. 数据擦除流程
      · 实现安全数据删除
      · 删除确认回执

  12. 备份恢复演练
      · 每月一次恢复演练
      · 记录演练结果
```

#### 步骤 7：整改验证

```
整改完成后：
  1. 自查所有 73 个控制点
  2. 确保满足率 ≥ 80%（三级要求）
  3. 准备整改报告
  4. 提交测评机构复核
```

### 第四阶段：正式测评（2-4 周）

#### 步骤 8：现场测评

```
测评机构现场检查：

  第 1 天：文档审查
    · 检查 12 个安全管理制度
    · 检查安全组织架构
    · 检查人员培训记录

  第 2-3 天：技术测试
    · 网络扫描和渗透测试
    · 主机安全检查
    · 应用安全测试
    · 数据安全检查

  第 4 天：访谈和验证
    · 访谈安全负责人
    · 验证应急响应流程
    · 检查审计日志

  第 5 天：出具测评报告
    · 通过 / 不通过
    · 整改建议（如不通过）
```

#### 步骤 9：获取测评报告

```
测评通过后：
  1. 获得《信息系统安全等级保护测评报告》
  2. 报告有效期：1 年
  3. 每年需要复评

测评不通过：
  1. 根据报告整改
  2. 重新提交测评（复测费用通常为首次的 50%）
  3. 整改后再次测评
```

### 第五阶段：持续合规（年度）

#### 步骤 10：年度复评

```
每年需要：
  1. 进行一次等保复评
  2. 复评费用：首次测评费的 50-70%
  3. 复评周期：2-3 周
  4. 复评内容：检查安全措施是否持续有效
```
