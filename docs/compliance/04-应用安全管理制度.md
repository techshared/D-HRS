# 应用安全管理制度

## 1. 身份认证
1. 用户访问系统必须经过身份认证
2. 支持实名认证（身份证 + 人脸识别）
3. 会话 Token（JWT）有效期不超过 24 小时

## 2. 访问控制
1. 基于角色的访问控制（RBAC），定义角色：HR_ADMIN、MANAGER、EMPLOYEE、AUDITOR
2. 每个角色仅拥有完成工作所需的最小权限
3. 智能合约层使用 OpenZeppelin AccessControl 实现链上权限控制

## 3. 安全审计
1. 所有用户操作记录详细日志（操作人、时间、IP、操作内容）
2. 智能合约中的关键操作（注册、审批、投票）均触发链上事件
3. 链下操作日志存储在数据库中，保留 ≥ 180 天

## 4. 应用安全防护
1. 所有 API 接口实施输入验证和参数过滤
2. 使用 Helmet 中间件设置安全 HTTP 头
3. 配置 CORS 策略，仅允许受信任的域名访问

## 5. 会话管理
1. 用户登出后立即销毁会话 Token
2. 支持单设备登录，新登录会踢出旧会话
3. 长时间无操作（≥ 30 分钟）自动退出登录

## 6. 智能合约安全
1. 合约代码经由安全审计后方可部署
2. 关键合约具备 Pausable 紧急停止功能
3. 使用 OpenZeppelin 标准库确保基础组件安全性
