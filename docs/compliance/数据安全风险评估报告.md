# 数据安全风险评估报告

**评估日期：** 2026年5月12日
**评估范围：** D-HRS（链聘通）全系统
**评估人：** D-HRS 开发团队

## 一、数据资产清单

| 数据类型 | 敏感等级 | 存储位置 | 加密方式 | 保留期限 |
|---------|---------|---------|---------|---------|
| 身份证号 | L4 极高 | 仅链下数据库 | SHA-256 哈希 + AES-256 | 员工离职后 6 个月 |
| 薪资信息 | L4 极高 | 仅链下数据库 | AES-256 | 员工离职后 6 个月 |
| 人脸数据 | L4 极高 | 仅链下数据库 | AES-256 | 认证完成后即删 |
| 姓名/手机号/邮箱 | L3 高 | 链上哈希 + 链下数据库 | 链下 AES-256 | 员工离职后 6 个月 |
| 职务/部门 | L2 内部 | 链上明文 + 链下数据库 | 无需加密 | 永久（审计需要） |
| 凭证数据 | L2 内部 | 链上哈希 + 链下数据库 | 链下 AES-256 | 凭证有效期 + 6 个月 |
| 操作日志 | L1 公开 | 链下数据库 | 明文 | 180 天 |
| 合约地址 | L1 公开 | 链上 + 配置文件 | 明文 | 永久 |

## 二、风险识别

### 2.1 数据泄露风险

| 风险项 | 可能性 | 影响 | 风险等级 | 缓解措施 |
|--------|-------|------|---------|---------|
| 链上数据永久可见 | 中 | 高 | 🔴 高 | 薪资/身份证不上链，仅存哈希 |
| 数据库被入侵 | 低 | 高 | 🟡 中 | AES-256 加密敏感字段 |
| API 未授权访问 | 中 | 高 | 🔴 高 | JWT 认证 + RBAC |
| 传输过程窃听 | 低 | 中 | 🟡 中 | TLS 1.3 加密 |
| 内部人员泄露 | 中 | 高 | 🔴 高 | 操作日志审计 + 最小权限 |

### 2.2 数据篡改风险

| 风险项 | 可能性 | 影响 | 风险等级 | 缓解措施 |
|--------|-------|------|---------|---------|
| 链上数据篡改 | 极低 | 高 | 🟢 低 | 区块链不可篡改特性 |
| 链下数据篡改 | 中 | 高 | 🟡 中 | 数据库审计日志 |
| 合约升级攻击 | 低 | 高 | 🟡 中 | TimelockController |

### 2.3 数据丢失风险

| 风险项 | 可能性 | 影响 | 风险等级 | 缓解措施 |
|--------|-------|------|---------|---------|
| 数据库丢失 | 低 | 高 | 🟡 中 | 每日备份 + 异地存储 |
| 链上数据丢失 | 极低 | 高 | 🟢 低 | 多节点同步 |
| 配置文件丢失 | 低 | 中 | 🟡 中 | Git 版本控制 |

## 四、安全措施清单

### 4.1 已实施

| 措施 | 说明 |
|------|------|
| 字段级加密 | 身份证号 SHA-256 + AES-256，薪资 AES-256 |
| 传输加密 | TLS 1.3（Helmet 强制） |
| 访问控制 | JWT 认证 + RBAC |
| 内容审核 | 敏感词过滤（员工信息 + 治理提案） |
| 操作日志 | 所有敏感操作记录审计日志 |
| 制裁筛查 | OFAC + 中国外交部名单 |
| 实名认证 | KYC 必须通过才能使用系统 |
| 用户同意 | 首次使用必须同意隐私政策和用户协议 |

### 4.2 待实施（需付费）

| 措施 | 费用 | 优先级 |
|------|------|--------|
| 等保三级测评 | ¥100K | 高 |
| 第三方安全审计 | ¥200K | 中 |
| 入侵检测系统 | ¥10K/年 | 中 |
| 数据库审计插件 | ¥5K/年 | 低 |

## 五、结论

D-HRS 系统在数据安全方面已实施了大部分免费可行的措施：
- ✅ 数据分级分类
- ✅ 敏感字段加密
- ✅ 用户同意机制
- ✅ 用户权利实现（查阅/更正/删除）
- ✅ 内容审核
- ✅ 操作日志审计
- ✅ 传输加密
- ✅ 访问控制

**当前数据安全合规度：约 85%**

剩余 15% 需要付费实施（等保测评 + 第三方审计 + 安全设备）。
