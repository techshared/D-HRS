# 安全运维管理制度

## 1. 日常巡检
1. 每日检查系统运行状态（CPU、内存、磁盘、网络）
2. 每日检查安全设备告警日志
3. 巡检结果记录在案，异常情况立即上报

## 2. 漏洞管理
1. 每月使用漏洞扫描工具对系统进行扫描
2. 高危漏洞 72 小时内完成修复
3. 中危漏洞 2 周内完成修复
4. 建立漏洞跟踪台账，直至修复完成

## 3. 补丁管理
1. 操作系统和应用软件及时更新安全补丁
2. 关键安全补丁在发布后 48 小时内完成测试和部署
3. 普通补丁在月度维护窗口统一部署

## 4. 日志管理
1. 系统日志类型包括：操作日志、安全日志、审计日志
2. 日志格式统一（时间戳、来源IP、操作用户、操作内容、结果）
3. 日志保留 ≥ 180 天
4. 日志存储空间不足时触发告警

## 5. 资产管理
1. 建立信息资产台账（硬件、软件、数据、账号）
2. 资产变更及时更新台账
3. 每年进行一次资产盘点

## 6. 第三方运维
1. 与运维服务商签署 SLA（服务水平协议）
2. 服务商人员按"最小必需"原则授权
3. 定期评估服务商安全能力
