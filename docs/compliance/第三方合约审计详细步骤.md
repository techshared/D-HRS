## 四、审计流程

### 步骤 1：联系审计公司（1-3 天）

```
联系方式：
  SlowMist（慢雾科技）
  · 官网：https://slowmist.com
  · 邮箱：service@slowmist.com
  · 微信：SlowMist（公众号）

沟通内容：
  1. 项目简介：D-HRS 链聘通
  2. 审计范围：9 个 Solidity 合约
  3. 代码行数：约 1,500 行
  4. 期望周期：2-3 周
  5. 预算范围：¥100K-200K
  6. 特殊要求：中国合规审查
```

### 步骤 2：签订合同（1 周）

```
合同内容：
  1. 审计范围：9 个合约
  2. 审计标准：行业最佳实践
  3. 审计费用：[与公司协商]
  4. 审计周期：[通常 2-3 周]
  5. 复测费用：首次不通过后的复测费用
  6. 报告格式：PDF 报告
  7. 保密条款：审计期间代码保密

合同签订后：
  · 支付定金（通常 50%）
  · 提交代码和文档
  · 开始审计
```

### 步骤 3：代码提交（1-2 天）

```
提交内容：
  1. 合约源码（contracts/src/*.sol）
  2. 测试用例（contracts/test/*.sol）
  3. 部署脚本（contracts/script/*.sol）
  4. 系统架构文档
  5. 合约功能规格说明
  6. 权限矩阵
  7. 威胁模型分析
  8. 自审计报告（SECURITY_AUDIT.md）

提交方式：
  · 私有 Git 仓库（推荐）
  · 加密压缩包
  · 安全文件传输
```

### 步骤 4：审计执行（2-3 周）

```
审计方法：
  1. 静态分析（Slither、Mythril）
  2. 人工代码审查
  3. 功能测试验证
  4. 安全漏洞扫描
  5. 业务逻辑审查

审计重点：
  · 重入攻击
  · 整数溢出
  · 访问控制
  · 逻辑漏洞
  · Gas 优化
  · 合规性检查

审计过程中：
  · 审计公司可能会提出问题
  · 需要及时回复
  · 可能需要补充文档
```

### 步骤 5：报告出具（3-5 天）

```
审计报告内容：
  1. 审计摘要
  2. 审计范围
  3. 审计方法
  4. 发现的问题（按严重程度分类）
     · Critical（严重）
     · High（高危）
     · Medium（中危）
     · Low（低危）
     · Informational（信息）
  5. 修复建议
  6. 审计结论

报告格式：
  · PDF 文档
  · 通常 20-50 页
  · 包含代码片段和截图
```

### 步骤 6：问题修复（1-2 周）

```
收到报告后：
  1. 按严重程度排序修复
  2. Critical 和 High 必须修复
  3. Medium 建议修复
  4. Low 可选修复
  5. 修复后提交复测

修复优先级：
  🔴 Critical：立即修复（24 小时内）
  🟡 High：尽快修复（1 周内）
  🟢 Medium：计划修复（2 周内）
  🔵 Low：可选修复（下次升级）
```

### 步骤 7：复测确认（3-5 天）

```
复测内容：
  1. 验证 Critical 和 High 问题已修复
  2. 验证 Medium 问题已修复（如已修复）
  3. 确认无新引入的问题
  4. 出具最终审计报告

最终报告：
  · 包含修复确认
  · 包含最终审计结论
  · 可用于备案和客户展示
```
