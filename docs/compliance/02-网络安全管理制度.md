# 网络安全管理制度

## 1. 网络架构安全
1. D-HRS 系统部署于云服务平台，采用虚拟私有云（VPC）进行网络隔离
2. 生产环境与开发环境严格分离，使用不同的 VPC 或账号
3. 系统对外仅开放必要的端口（HTTPS 443、API 3001）
4. 内部服务间通信使用内网地址，不暴露于公网

## 2. 访问控制
1. 实施网络边界访问控制策略，默认拒绝所有入站流量
2. 仅允许白名单 IP 地址访问管理端口
3. 所有 API 请求必须经过身份认证中间件

## 3. 入侵检测与防护
1. 启用云平台安全组和 Web 应用防火墙（WAF）
2. 配置入侵检测策略，对异常流量进行告警
3. 定期分析安全日志，发现潜在威胁

## 4. 安全监控
1. 对网络流量进行实时监控
2. 配置网络异常行为告警规则
3. 保留网络日志不少于 180 天

## 5. 无线安全
1. 运维终端连接企业 WiFi 时使用 WPA2 加密
2. 禁止使用非加密的公共 WiFi 进行系统运维
