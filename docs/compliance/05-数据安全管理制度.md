# 数据安全管理制度

## 1. 数据分类分级
D-HRS 数据分为以下等级：
- **L4 极高敏感**：身份证号、人脸信息、薪资数据 → 加密存储 + 访问审批
- **L3 高敏感**：员工姓名、手机号、邮箱 → 加密存储
- **L2 内部**：职务信息、部门、绩效评语 → 内部授权访问
- **L1 公开**：合约地址、系统版本号 → 可公开

## 2. 数据加密
1. 静态数据：数据库存储使用 AES-256 加密
2. 传输数据：所有 API 通信使用 TLS 1.3
3. 链上数据：敏感信息仅存储哈希值，明文不上链
4. 密钥管理：加密密钥定期轮换（每 90 天），使用硬件安全模块（HSM）或云 KMS

## 3. 数据访问控制
1. 数据访问遵循最小权限原则
2. 对 L4 级数据的访问需双重审批
3. 所有数据访问记录审计日志

## 4. 数据备份
1. 数据库每日全量备份，每 4 小时增量备份
2. 备份数据异地存储，与生产中心距离 ≥ 50 公里
3. 每月进行一次备份恢复演练

## 5. 数据销毁
1. 员工离职后，个人信息保留期限不超过法律要求的最短时间
2. 数据销毁使用安全擦除方法，确保不可恢复
3. 链上数据因技术限制无法删除，通过撤销访问权限实现"功能删除"

## 6. 个人信息保护（《个人信息保护法》合规）
1. 收集个人信息前获取用户明示同意
2. 提供个人信息查阅、更正、删除渠道
3. 不超范围收集个人信息
4. 个人信息出境前完成安全评估
