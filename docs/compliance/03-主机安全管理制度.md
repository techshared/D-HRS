# 主机安全管理制度

## 1. 身份鉴别
1. 所有服务器使用强密码策略：长度 ≥ 12 位，含大小写字母、数字和特殊字符
2. 启用 SSH 密钥认证，禁止密码登录
3. 每 90 天更换一次密钥对

## 2. 访问控制
1. 操作系统遵循最小权限原则
2. 应用服务使用专用运行账户，禁止以 root 运行
3. sudo 权限仅授予运维人员，记录所有 sudo 操作

## 3. 安全审计
1. 启用系统审计日志（auditd），记录用户登录、命令执行等关键事件
2. 审计日志保留 ≥ 180 天
3. 定期审查审计日志，发现异常行为及时处置

## 4. 入侵防范
1. 安装并配置主机入侵检测系统（HIDS）
2. 每月对系统进行漏洞扫描
3. 及时安装安全补丁，关键补丁在 72 小时内完成

## 5. 资源控制
1. 配置进程资源限制（ulimit），防止资源耗尽
2. 实施会话超时策略，闲置 15 分钟自动退出
3. 限制同一用户的最大并发会话数
