## 论文骨架（完整大纲）

### 1. 引言（约1500字）

**1.1 背景**
- 中国企业HR数字化转型趋势（北森/Moka等SaaS渗透率约20%，增长25% CAGR）
- HR合规审计的法规环境：924通知、PIPL、数据安全法、等保2.0
- 企业在HR审计中的痛点：操作不可追溯、资质文件造假、等保评审难通过

**1.2 问题提出**
- 传统HR系统的审计缺陷：中心化数据库的可篡改性问题
- 区块链存证的优势：不可篡改、时间戳、去信任化验证
- 现有区块链HR系统的问题（发币风险、隐私违规、合规缺失）

**1.3 贡献**
1. 设计了一套**符合中国监管环境**的HR数据存证架构（不上链隐私数据、不发币）
2. 实现了一个**双层合规守卫机制**（合约层+中间件层双重KYC/制裁检查）
3. 完整的等保三级管理制度体系和区块链备案材料
4. 开源实现并验证了35个合约测试用例

### 3. 系统设计（约3000字 — 核心）

**3.1 设计原则**
1. **合规优先**：所有设计以中国法规为约束条件
2. **隐私保护**：不上链任何个人敏感信息（PIPL第6条）
3. **零加密货币**：无ERC-20发行、无链上支付（924通知）
4. **分层架构**：合约层 + 中间件层分离，便于审计

**3.2 系统架构**
```
┌──────────────────────────────────────────────┐
│              应用层（Web/Mobile）              │
├──────────────────────────────────────────────┤
│               API网关（Express.js）            │
│  [JWT认证] [KYC中间件] [制裁筛查] [内容审核]    │
├──────────────────────────────────────────────┤
│            智能合约层（Solidity 0.8.19）        │
│  ComplianceEngine → guard()                    │
│  EmployeeRegistry → DID + 角色 + 状态          │
│  CredentialRegistry → VC哈希存证               │
│  PayrollExecutor → 多签审批流                  │
└──────────────────────────────────────────────┘
```

**3.3 合规引擎设计（重点贡献）**
- `guard()` 函数设计：在每次写入操作前执行KYC状态检查 + 制裁名单检查
- 双层合规：合约层 `require()` + 后端中间件同步拦截
- 限频机制：100次/地址/天，1分钟冷却

**3.4 隐私保护设计**
- AES-256字段级加密（身份证号、薪资数据）
- 链上仅存 `hash(data)` + `metadata_uri`
- 用户同意机制（首次使用需签署用户协议+隐私政策）

**3.5 智能合约关键实现**
- EmployeeRegistry：DID注册（不上链工资）
- PayrollExecutor：多签审批（不上链支付，无ERC-20）
- BenefitsNFT：Soulbound（`_beforeTokenTransfer`限制转账）
- DecentralizedHRS：招聘/评估/晋升/离职全流程

### 4. 系统实现（约1500字）

**4.1 开发环境**
- Solidity 0.8.19 + Foundry（测试框架）
- Node.js 20 + Express 4.18
- Hardhat 私链（兼容以太坊EVM）

**4.2 关键实现细节**
- OpenZeppelin AccessControl 角色管理（HR_ADMIN, FINANCE, EXECUTOR, VOTER_ROLE）
- Pausable 紧急停止机制
- ReentrancyGuard 重入保护
- 内容审核（敏感词过滤 + 人工复核通道）

**4.3 开源仓库**
- 9个合约，35个单元测试，全部通过
- 3个关键安全漏洞已修复（详见SECURITY_AUDIT.md）
- 12份等保三级管理制度文档
- 区块链备案材料
