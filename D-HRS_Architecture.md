## 2. Mutual Authentication Flow

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                    MUTUAL AUTHENTICATION (mTLS) FLOW                            │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                  │
│   CLIENT (Employee/HR)                    SERVER (D-HRS Node)                   │
│   ┌──────────────┐                       ┌──────────────┐                       │
│   │  Browser/    │                       │  API         │                       │
│   │  Application │                       │  Gateway     │                       │
│   └──────┬───────┘                       └──────┬───────┘                       │
│          │                                        │                               │
│          │     1. Client Hello                    │                               │
│          │  ┌─────────────────────────────────┐  │                               │
│          │  │ • Supported cipher suites       │  │                               │
│          │  │ • Client random number          │  │                               │
│          │  │ • Client certificate request    │──┼──►                            │
│          │  └─────────────────────────────────┘  │                               │
│          │                                        │                               │
│          │                                        │     2. Server Hello           │
│          │     3. Server Certificate             │  ┌─────────────────────────┐ │
│          │  ┌─────────────────────────────────┐  │  │ • Server certificate    │ │
│          │  │ • Server public key            │◄─┼──│ • Server random          │ │
│          │  │ • Server random                │  │  │ • Selected cipher suite │ │
│          │  │ • Certificate chain           │  │  │ • CA signed certificate │ │
│          │  └─────────────────────────────────┘  │  └─────────────────────────┘ │
│          │                                        │                               │
│          │     4. Client Certificate              │                               │
│          │  ┌─────────────────────────────────┐  │                               │
│          │  │ • Employee/HR wallet cert     │──┼──►                            │
│          │  │ • Signed by Organization CA    │  │                               │
│          │  │ • Contains blockchain identity │  │                               │
│          │  └─────────────────────────────────┘  │                               │
│          │                                        │                               │
│          │                                        │     5. Verify Client Cert    │
│          │                                        │  ┌─────────────────────────┐ │
│          │                                        │  │ • Check CA signature    │ │
│          │                                        │  │ • Verify not expired    │ │
│          │                                        │  │ • Check CRL/OCSP        │ │
│          │                                        │  │ • Validate blockchain   │ │
│          │                                        │  │   DID/VC binding        │ │
│          │                                        │  └─────────────────────────┘ │
│          │                                        │                               │
│          │     6. Pre-Master Secret (Encrypted)  │                               │
│          │  ┌─────────────────────────────────┐  │                               │
│          │  │ • Encrypted with server pubkey │──┼──►                            │
│          │  └─────────────────────────────────┘  │                               │
│          │                                        │                               │
│          │     7. Session Keys Generated         │                               │
│          │  ┌─────────────────────────────────┐  │                               │
│          │  │ • Master secret                 │  │                               │
│          │  │ • Client write key              │◄─┼──► Both generate            │
│          │  │ • Server write key              │  │      matching keys          │
│          │  │ • Client MAC key                │  │                               │
│          │  │ • Server MAC key                │  │                               │
│          │  └─────────────────────────────────┘  │                               │
│          │                                        │                               │
│          │     8. Finished (Encrypted + MAC)    │                               │
│          │  ┌─────────────────────────────────┐  │                               │
│          │  │ • Verify all handshake messages │──┼──►                            │
│          │  │ • Hash of all handshake data    │  │                               │
│          │  └─────────────────────────────────┘  │                               │
│          │                                        │                               │
│          │                                        │     9. Finished               │
│          │  ┌─────────────────────────────────┐  │                               │
│          │  │ • Verify server authenticity   │◄─┼──►                            │
│          │  └─────────────────────────────────┘  │                               │
│          │                                        │                               │
│          ══════════════════════════════════════════════════════                 │
│                              SECURE CHANNEL ESTABLISHED                        │
│          ══════════════════════════════════════════════════════                 │
│                                                                                  │
│  ┌─────────────────────────────────────────────────────────────────────────┐    │
│  │              BLOCKCHAIN IDENTITY VERIFICATION                           │    │
│  │                                                                          │    │
│  │   Certificate ←→ DID Binding:                                          │    │
│  │   ┌──────────────────────────────────────────────────────────────────┐   │    │
│  │   │  X.509 Cert Subject:                                            │   │    │
│  │   │  CN = did:hrs:abc123#keys-1                                     │   │    │
│  │   │                                                                  │   │    │
│  │   │  Blockchain Query:                                              │   │    │
│  │   │  Resolve DID "did:hrs:abc123" → Employee Identity Record        │   │    │
│  │   │                                                                  │   │    │
│  │   │  Verify:                                                        │   │    │
│  │   │  • DID owner matches certificate holder                        │   │    │
│  │   │  • Credentials valid (not revoked)                             │   │    │
│  │   │  • Employment status active                                     │   │    │
│  │   └──────────────────────────────────────────────────────────────────┘   │    │
│  └─────────────────────────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────────────────────────┘
```

## 8. Technology Stack Summary

| Layer | Technology |
|-------|------------|
| **Frontend** | React, Vue.js, React Native |
| **API Gateway** | Kong, NGINX, Envoy |
| **Authentication** | mTLS, OAuth 2.0, OIDC, DID/VC |
| **Blockchain** | Hyperledger Besu, Quorum, or Custom |
| **Smart Contracts** | Solidity, Vyper |
| **Identity** | DID, W3C Verifiable Credentials |

| **Monitoring** | Prometheus, Grafana, ELK Stack |

---

This architecture provides:
- **Mutual Authentication**: Both client and server verify each other's identity
- **Blockchain-Integrated Identity**: DID/VC binding with X.509 certificates
- **Defense in Depth**: 7-layer security model
- **Decentralized Trust**: No single point of failure
- **Immutable Audit Trail**: All actions recorded on blockchain
